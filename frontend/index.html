Heart Disease Prediction App

Academic demonstration only. This tool is not a medical diagnosis system and must not be used for clinical decisions.

Abstract

Abstract

Cardiovascular disease remains the leading cause of mortality worldwide, placing a growing burden on healthcare systems. Early and accurate assessment of heart-disease severity is critical for guiding treatment and improving patient outcomes. In this work, we present a case study on predicting heart-disease severity using machine-learning algorithms applied to a publicly available dataset. Through an exploratory data analysis we characterize clinical features, perform data cleaning, and engineer novel predictors, generating several tailored datasets to train our predictive models. Our six supervised learning models—Logistic Regression, k-Nearest Neighbors, Naive Bayes, Support Vector Machines, Random Forests, and XGBoost—are trained and evaluated via stratified 5-fold cross-validation in both binary (disease vs. no disease) and five-class severity (0–4) settings. Performance is measured using accuracy, precision, recall, F1-score, and area under the ROC curve. Our best model, Random Forest, achieves an overall accuracy of 83%, macro average recall of 82%, and an F1-score of 82%, outperforming baseline classifiers by a substantial margin. Feature-importance analysis identifies [ST depression, maximum heart rate, age] as the strongest predictors of severity. These results demonstrate the promise of ensemble and decision-tree techniques for automated heart-disease assessment.

Data collecting, cleaning and preprocessing

Exploratory Data Analysis, cleaning and dataset construction

The practical chapter starts from a public heart-disease dataset and builds the data foundation needed for model comparison. The goal is not only to clean the data, but to reason step by step about which information can be trusted, which missing values can be imputed, and which design choices preserve enough signal to outperform a dummy classifier later.

1. Dataset source and target definition

The original file, heart_disease_uci.csv, contains 920 patient records collected from four sources: Cleveland, Hungary, Switzerland and VA Long Beach. Each record combines demographic variables, laboratory measurements, resting ECG information and exercise-test results. The target variable is num, assigned after clinical evaluation.

This distribution immediately shapes the project. A naive multiclass model can obtain apparently acceptable accuracy by overpredicting class 0, while failing to identify the clinically important minority stages. For that reason, the thesis tracks recall and macro-averaged metrics rather than relying only on accuracy.

2. Feature inventory

The dataset contains one patient identifier (id), fourteen predictor variables, and one target variable. The predictors are divided into continuous and categorical variables as follows:

  • id (identifier): unique patient number used to distinguish each record. This variable is not used as a predictive feature.
  • age (continuous): patient age in years.
  • sex (categorical): patient sex, recorded as male or female.
  • dataset (categorical): location where the data were collected: Cleveland, Hungary, VA Long Beach, or Switzerland.
  • cp (categorical): type of chest pain reported by the patient: typical angina, atypical angina, non-anginal pain, or asymptomatic.
  • trestbps (continuous): resting blood pressure, measured in mmHg on admission.
  • chol (continuous): serum cholesterol level, measured in mg/dL.
  • fbs (categorical): whether fasting blood sugar is greater than 120 mg/dL, recorded as true or false.
  • restecg (categorical): resting electrocardiogram result, classified as normal, ST-T wave abnormality, or left ventricular hypertrophy.
  • thalach (continuous): maximum heart rate achieved during exercise.
  • exang (categorical): presence of exercise-induced angina, recorded as true or false.
  • oldpeak (continuous): ST depression induced by exercise relative to rest, measured in mm.
  • slope (categorical): slope of the peak exercise ST segment, classified as upsloping, flat, or downsloping.
  • ca (categorical): number of major vessels, from 0 to 3, colored by fluoroscopy.
  • thal (categorical): thallium stress test result, classified as normal, fixed defect, or reversible defect.

The target variable is:

  • num (categorical): diagnostic label assigned after angiographic evaluation. A value of 0 indicates no presence of heart disease, while values from 1 to 4 represent increasing severity, from mild to severe heart disease.

This feature inventory is important because preprocessing depends on variable type. Categorical variables are later transformed using one-hot encoding, while continuous variables are standardized, especially for distance-sensitive models such as k-NN and SVM.

3. First cleaning: impossible physiological values

The EDA does not treat all numeric values as valid just because they are present. Two impossible values are detected: one patient has trestbps = 0 mmHg and 172 records have chol = 0 mg/dL. Both are physiologically impossible in living patients, so they are converted to missing values before any distributional analysis or model fitting. 

df["trestbps"] = df["trestbps"].replace(0, np.nan)
                df["chol"] = df["chol"].replace(0, np.nan)

This decision prevents erroneous zeros from pulling means, histograms and downstream imputers toward impossible values.

4. Missing-value map and row-level deletion

Before deleting anything, missing values are counted both by feature and by row. The most problematic variables are ca, thal and slope, with missingness of 66.41%, 52.83% and 33.59% in the original dataset after physiological cleaning. The row-level view shows that missingness is concentrated in a subset of highly incomplete records.

5. Three dataset-construction strategies

After row filtering, the project deliberately constructs three datasets instead of betting on one preprocessing assumption. This allows the training notebook to measure how different missing-value strategies affect accuracy, recall and class balance.

6. Imputation logic

Simple and predictive imputations are used only where they are defensible. trestbps and oldpeak have only one missing value after row filtering, so mean imputation has negligible distributional impact. fbs is binary and imputed by sampling according to its observed frequency distribution. chol still has 13.15% missingness, so several predictive imputers are tested; because they do not beat the mean baseline, mean imputation is selected.

The high-missingness variables require stronger decisions. ca is dropped because 59.77% missingness remains after filtering. slope and thal are clinically meaningful, so they are collapsed into less sparse binary indicators: slope_bin is flat vs non-flat, and thal_bin is defect vs no defect. This sacrifices category granularity to preserve usable signal. 

df_clean["slope_bin"] = df_clean["slope"].map({
                "flat": "flat",
                "upsloping": "non-flat",
                "downsloping": "non-flat"
                })

                df_clean["thal_bin"] = df_clean["thal"].map({
                "normal": "no defect",
                "fixed defect": "defect",
                "reversable defect": "defect"
})

7. Relationship analysis

Correlation analysis is used to understand which variables carry signal about severity. For numeric features, Pearson correlation identifies oldpeak, thalach and age as the most informative predictors. Higher ST depression and lower maximum heart rate align with more severe heart disease, which is clinically plausible.

For categorical variables, the analysis uses an ANOVA-style correlation ratio. Chest pain type and exercise-induced angina are the strongest categorical predictors across the processed datasets, followed by thalassemia status, sex and resting ECG.

8. Outlier detection and treatment

Because clinical datasets can contain legitimate extreme measurements, the project avoids the standard 25th-75th percentile IQR rule, which would be too aggressive for a modest sample. Instead, it computes a wider 5th-95th percentile IQR band and removes only rows outside that relaxed range.

This removes 2 outliers from the No NaNs dataset and 1 outlier from each imputed dataset, preserving nearly all usable data while protecting models from extreme leverage points.

9. Final Datasets

At the end of the exploratory data analysis, the project has constructed three final datasets for the model training stage. These datasets represent different preprocessing assumptions, allowing the training notebook to compare whether complete-case deletion, imputation with variable removal, or imputation with collapsed clinical indicators produces the strongest predictive performance.

These three datasets are the final inputs used for the model-training experiments: the No NaNs dataset, the No Ill Conditions dataset, and the Ill Conditions dataset.

Training protocol, evaluation metrics and final model choice

1. Why nested cross-validation?

A single train/test split would make the result depend too much on one random partition. The thesis therefore uses a 5-fold outer cross-validation loop and a 3-fold inner loop for hyperparameter tuning. The outer test fold is never used to choose hyperparameters, so the final score is a less biased estimate of generalization. 

outer_cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
              grid = GridSearchCV(model, param_grid, cv=3, scoring="recall")
              y_pred = cross_val_predict(grid, X, y, cv=outer_cv)

Six supervised classifiers are compared: Logistic Regression, k-Nearest Neighbors, Random Forest, Naive Bayes, SVM with RBF kernel, and XGBoost. A Dummy Classifier is kept as the minimum baseline to beat. Hyperparameters are chosen to span simple and more flexible decision boundaries while keeping the grid small enough for repeated cross-validation.

3. Binary classifier: disease vs no disease

The first modelling target collapses num into a binary label: 0 means no disease and 1 means any disease stage. This is a clinically useful screening task and provides a baseline before attempting severity staging.

On the Ill Conditions dataset, k-NN achieves the strongest practical balance: approximately 0.84 accuracy and 0.86 recall for the disease class. The dummy classifier reaches only 0.53 accuracy on this dataset by always predicting disease, so the tuned model is meaningfully better than the baseline.

4. Multiclass severity prediction

The full target num ∈ {0,1,2,3,4} is harder because the original class distribution is highly skewed. On the original datasets, most models obtain only around 0.50-0.60 accuracy and weak macro recall, meaning they mostly recognize the majority class and fail on rarer severe stages.

To address this, the notebook constructs four resampled variants: Balanced, Equalized to Class 1, Equalized to Class 2, and Equalized to Class 4. Equalizing to Class 1 gives the strongest multiclass result because it preserves more total samples while making all classes equally represented. Random Forest and XGBoost reach approximately 0.83 accuracy and 0.83 macro recall on that dataset.

5. Two-step classifier

Because direct five-class prediction is unstable on the original distribution, the thesis also evaluates a cascaded approach. Step 1 detects whether disease is present. Step 2, applied only to predicted diseased patients, separates Stage 1 from Stages 2-3-4. This simplifies each decision boundary and focuses the second model on clinically meaningful progression.

Logistic Regression on the Ill Conditions binary subset reaches about 0.75 accuracy and 0.76 macro recall for the early-vs-advanced stage task. It correctly identifies roughly 84% of advanced cases, which is the desired direction for a sensitivity-oriented medical workflow.

6. Why the web demo uses the binary Ill Conditions + k-NN model

The web application is meant to be a reliable academic demonstration, not a clinical staging tool. The binary task is the most stable on the original processed datasets, and the Ill Conditions dataset keeps useful transformed clinical variables without relying on the heavily missing original ca, slope and thal fields. The persisted model therefore uses modified_dataset_with_imputations.csv, notebook-style dummy encoding, mean imputation, standard scaling and tuned k-NN with k = 13.
